## 1. 进程线程模型

WebUI 启动后并存三类线程，职责分离：

```mermaid
flowchart TD
    MAIN["entry.py → server.main()"]
    MAIN --> T1["主线程<br/>ThreadingHTTPServer.serve_forever()<br/>处理 HTTP 请求"]
    MAIN --> T2["调度线程<br/>WebScheduler._run()<br/>每5分钟提交维护任务"]
    T2 --> T3["任务线程池<br/>WebTaskStore (max 4)<br/>执行 run_auto_maintenance 等"]
    T1 -.->|HTTP 触发| T3
    note["调度器只'按时提交'，<br/>维护逻辑跑在线程池，<br/>不阻塞调度节拍与 HTTP"]
```

## 2. HTTP 请求处理与认证

```mermaid
sequenceDiagram
    autonumber
    participant B as 浏览器
    participant H as WebUIHandler
    participant ST as WebState
    participant API as dispatch_api

    B->>H: POST /api/xxx (Cookie + X-CSRF-Token)
    H->>H: _is_authorized() 校验 session∈sessions
    alt 未认证
        H-->>B: 401 unauthorized
    end
    H->>H: _check_csrf() 校验 X-CSRF-Token==csrf_token
    alt CSRF 失败
        H-->>B: 403
    end
    H->>API: dispatch_api(path, payload, state)
    API->>ST: 读/写配置、提交任务
    API-->>H: result
    H-->>B: {ok:true, result}
```

登录：`POST /login` 校验 `password==auth_secret` → 生成 session_id 写入 `sessions` → 下发 `Set-Cookie: sub2api_web_session=...; HttpOnly; SameSite=Lax`。

## 3. 端到端运营全景（核心）

从进程启动到一轮完整发卡的全链路：

```mermaid
sequenceDiagram
    autonumber
    participant SCH as 调度线程
    participant POOL as 任务线程池
    participant AUTO as run_auto_maintenance
    participant ACC as acc 席位层
    participant OPENAI as OpenAI/ChatGPT
    participant S2A as Sub2API
    participant REG as 注册引擎
    participant OIDC as OIDC

    SCH->>SCH: 启动延迟10s / 周期300s 到点
    SCH->>POOL: tasks.create("auto_maintenance", run_auto_maintenance)
    POOL->>AUTO: 执行(state)

    rect rgb(235,245,255)
    Note over AUTO,OPENAI: Phase 1 低额度降级
    AUTO->>ACC: enforce_acc_low_quota_policy
    ACC->>S2A: load_sub2api_usage_lookup (读5h/7d额度)
    ACC->>OPENAI: list_all_users (母号成员)
    ACC->>S2A: set_remote_accounts_inactive (停用低额度)
    ACC->>OPENAI: PATCH 席位=usage_based (降级Codex)
    ACC->>OPENAI: enforce_chatgpt_seat_limit (收敛≤2)
    end

    rect rgb(235,255,235)
    Note over AUTO,S2A: Phase 2 扫描
    AUTO->>S2A: scan_remote_accounts (3线程拉取+并发校验)
    S2A->>OPENAI: 每账号 wham/usage (判活/额度)
    S2A-->>AUTO: {total,alive,dead,no_quota}
    end

    Note over AUTO: Phase 3 quota_ok = alive-no_quota
    alt quota_ok ≥ threshold (水位够)
        AUTO-->>POOL: 结束(skip补号)
    else 需补号
        rect rgb(255,245,235)
        Note over AUTO,OPENAI: Phase 4 注册
        AUTO->>REG: register_batch(count-quota_ok, 串行)
        REG->>OPENAI: CF绕过+Sentinel PoW+企业SSO+PKCE OAuth
        OPENAI-->>REG: refresh/access/id_token
        REG-->>AUTO: [RegisterResult(ok,token_json)]
        end

        rect rgb(245,235,255)
        Note over AUTO,S2A: Phase 5 导入
        loop 每个成功账号
            AUTO->>S2A: import_to_sub2api_codex_session
            S2A->>S2A: import + bulk-update(active,直通)
        end
        end

        Note over AUTO,OIDC: Phase 6 发卡
        AUTO->>OIDC: oidc_generate_cards(成功数, 30天)
        OIDC-->>AUTO: 卡密列表
    end
    AUTO-->>POOL: report{phases, pool_status, elapsed}
```

## 4. 调度器主循环

```mermaid
stateDiagram-v2
    [*] --> 初始化: 读配置, next_run=now+(10 or interval)
    初始化 --> 等待: wait(min(剩余,5s))
    等待 --> 检查停止: stop_event?
    检查停止 --> [*]: set
    检查停止 --> 检查唤醒: 未set
    检查唤醒 --> 重读配置: wake_event(配置变更)
    重读配置 --> 等待: next_run=now+min(10,interval)
    检查唤醒 --> 到点判断: 无唤醒
    到点判断 --> 等待: now<next_run
    到点判断 --> 提交任务: now≥next_run且配置齐全
    到点判断 --> 跳过: 配置缺失/已禁用
    提交任务 --> 重排: next_run=now+interval
    跳过 --> 重排
    重排 --> 等待
```

## 5. 自动注册单账号时序

```mermaid
sequenceDiagram
    autonumber
    participant R as register_one
    participant CG as chatgpt.com
    participant AU as auth.openai.com
    participant SEN as sentinel.openai.com

    R->>CG: GET / 预热 +随机延时
    R->>CG: GET /api/auth/csrf
    R->>CG: POST /api/auth/signin/openai → authorize URL
    R->>AU: GET authorize URL (同步 oai-did)
    alt about-you 普通创建
        R->>SEN: _build_sentinel(create_account) 求解PoW
        R->>AU: POST /api/accounts/create_account + sentinel
    else /sso 企业SSO免验证码
        R->>R: 从 cookie 取 SSO connection
        R->>SEN: _build_sentinel(authorize_continue)
        R->>AU: POST /api/accounts/authorize/continue
        R->>AU: 逐跳→ POST approve {email, confirm_password:"", challenge}
        R->>AU: POST consent/interstitial 表单
    end
    R->>CG: GET /api/auth/session (重试≤10) → accessToken
    R->>CG: GET /me + accounts/check → user_id, account_id
    R->>CG: PATCH onboarding(role=engineering)
    R->>AU: GET /oauth/authorize (PKCE) → code
    R->>AU: POST /oauth/token → access/refresh/id_token
    R->>R: build_codex_token_json
```

sequenceDiagram

    BR->>BR: 用户授权 → 302 localhost:1455?code=&state=

## 9. 账号资源生命周期

stateDiagram-v2

## 12. 桌面端后台任务框架

```mermaid
flowchart LR
    UI["用户操作"] --> ST["start_background_task"]
    ST --> BUSY["busy=True 禁用按钮"]
    ST --> TH["daemon Thread"]
    TH --> WK["worker() 网络请求"]
    WK -->|成功| OK["root.after(0, on_success)"]
    WK -->|异常| ER["root.after(0, 错误弹窗)"]
    OK --> FIN["busy=False 恢复按钮"]
    ER --> FIN
    Note over TH,FIN: 跨线程 GUI 更新一律 root.after 回主线程
```
