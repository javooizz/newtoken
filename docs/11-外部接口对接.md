# 11 · 外部接口对接

本项目对接三个外部系统。本文汇总所有外部 HTTP 接口的完整契约（URL / 方法 / 鉴权 / 请求 / 响应），供排障与二次开发查阅。

| 外部系统 | 鉴权 | 用途 |
|----------|------|------|
| **OpenAI / ChatGPT** | Bearer access_token / session cookie / sentinel token | 注册、席位管理、额度查询、token 续期、OAuth |
| **Sub2API 网关** | `x-api-key` 头 | 账号导入/扫描/删除/批量更新/OAuth 建号 |
| **OIDC 卡密服务** | `Authorization: Bearer` | 发卡/查卡/探活 |

## 1. OpenAI / ChatGPT 接口

### 1.1 自动注册链路（`webui/register.py`）

| 端点 | 方法 | 关键参数/头 | 用途 |
|------|------|------------|------|
| `chatgpt.com/` | GET | — | 预热拿 cookie |
| `chatgpt.com/api/auth/csrf` | GET | — | 取 csrfToken |
| `chatgpt.com/api/auth/signin/openai` | POST | `login_hint`、`screen_hint=login_or_signup`、csrfToken | 返回 authorize URL |
| `auth.openai.com/oauth/authorize`（首次） | GET | 跟随重定向 | 落地判分支，同步 oai-did |
| `sentinel.openai.com/backend-api/sentinel/req` | POST | `{p, id:device_id, flow}`，`Content-Type: text/plain` | 取 PoW 挑战 |
| `auth.openai.com/api/accounts/create_account` | POST | `{name, birthdate}` + `openai-sentinel-token` + Datadog 头 | 创建账号（普通路径） |
| `auth.openai.com/api/accounts/authorize/continue` | POST | `{connection, connection_provider}` + sentinel | 企业 SSO 流入口 |
| SSO approve / consent 表单 | POST | `{email, confirm_password:"", challenge}` / `{interstitial_token, csrf_token}` | SSO 自动批准 |
| `chatgpt.com/api/auth/session` | GET | cookie | 拿 accessToken（重试≤10） |
| `chatgpt.com/backend-api/me` | GET | Bearer | 取 user_id |
| `chatgpt.com/backend-api/accounts/check/v4-2023-04-27` | GET | Bearer | 取 account_id |
| `chatgpt.com/backend-api/accounts/{aid}/users/{uid}` | PATCH | `{onboarding_information:{role:engineering}}` | 新号 onboarding |
| `auth.openai.com/oauth/authorize`（Codex） | GET | PKCE `code_challenge`(S256)、`client_id=app_EMoam...`、`scope`(含 offline_access)、`originator=codex_vscode` | 拿授权 code |
| `auth.openai.com/oauth/token` | POST | `grant_type=authorization_code`、`code`、`code_verifier` | 换 access/refresh/id_token |

### 1.2 席位管理（`acc/seat_client.py` — accounts API）

鉴权：`authorization: Bearer <access_token>` 或 `cookie: __Secure-next-auth.session-token=...`；必带 `account-id`、`oai-client-build-number`、`oai-client-version`、`oai-device-id`、Windows Chrome UA。

| 端点 | 方法 | 请求 | 响应 |
|------|------|------|------|
| `/backend-api/accounts/{account_id}/users?offset=&limit=&query=` | GET | — | `{items:[{id,email,seat_type,...}], total}` |
| `/backend-api/accounts/{account_id}/users/{user_id}` | PATCH | `{seat_type: "default"\|"usage_based"}` | `{success:true}` |
| `/backend-api/accounts/{account_id}/invites` | POST | `{email_addresses, role, seat_type:"usage_based", resend_emails}` | `{errored_emails:[]}` |
| `/api/auth/session` | GET | cookie | `{accessToken, account:{id}}` |

### 1.3 组织管理（`acc/gpt_space_manager.py` — organizations API）

鉴权：纯 session cookie（`__Secure-next-auth.session-token`）+ `X-Csrf-Token`。

| 端点 | 方法 | 请求 | 用途 |
|------|------|------|------|
| `/backend-api/organizations/members` | GET | — | 拉成员（失败 fallback 探测 `/accounts/check/v4-2024-10-25`） |
| `/backend-api/organizations/members` | POST | `{email, seat_type:"codex"}` | 加成员 |
| `/backend-api/organizations/invites` | POST | `{email}` | fallback：发邀请拿 invite_id |
| `/backend-api/organizations/invites/{id}/assign-seat` | POST | `{seat_type}` | fallback：分配席位 |

### 1.4 额度与续期（`sub2api/converter_core.py`）

| 端点 | 方法 | 请求 | 响应 |
|------|------|------|------|
| `auth.openai.com/oauth/token` | POST | `{client_id:app_EMoam..., grant_type:refresh_token, refresh_token}` | `{access_token, id_token, refresh_token}` |
| `chatgpt.com/backend-api/wham/usage` | GET | `Bearer` + `ChatGPT-Account-Id` | `{rate_limit:{allowed, limit_reached, primary_window, secondary_window}}` |

`wham/usage` 的 `primary_window`(5h)/`secondary_window`(7d) 各含 `used_percent`、`reset_at`/`reset_after_seconds`、`limit_window_seconds`。

## 2. Sub2API 管理端 REST

**基础约定**：
- Base：`SUB2API_BASE_URL`（http/https，可带反代子路径，去尾斜杠）。
- 鉴权：HTTP 头 **`x-api-key: <SUB2API_ADMIN_API_KEY>`** + `Accept: application/json`。
- 响应壳：`{"code":0, "message":"...", "data":<object|list>}`，code=0/缺失为成功。

| # | 端点 | 方法 | 请求体关键字段 | 响应 data 字段 |
|---|------|------|----------------|----------------|
| 1 | `/api/v1/admin/accounts?page=1&page_size=1` | GET | — | `total`（连接测试） |
| 2 | `/api/v1/admin/accounts?platform=openai&type=oauth&page=&page_size=1000&sort_by=name&sort_order=asc` | GET | — | `items[{id,name,credentials,status,extra}]`、`total` |
| 3 | `/api/v1/admin/accounts/data?platform=openai&type=oauth&include_proxies=false` | GET | — | `accounts[]`（完整 credentials） |

| 18 | `/api/v1/admin/openai/create-from-oauth` | POST | `session_id,code,state,redirect_uri,name,concurrency,group_ids,proxy_id?` | `id,name,email` |

### 2.1 远程账号 item 结构

```jsonc
{
  "id": 123, "name": "user@domain", "status": "active",
  "credentials": {
    "access_token": "...", "refresh_token": "...", "id_token": "...",
    "email": "...", "chatgpt_account_id": "...", "chatgpt_user_id": "...",
    "chatgpt_plan_type": "...", "expires_at": "..."
  },
  "extra": {
    "codex_5h_used_percent": 15.0, "codex_7d_used_percent": 40.0,
    "codex_usage_updated_at": "...",
    "codex_5h_reset_at": "...", "codex_7d_reset_at": "...",
    "codex_5h_reset_after_seconds": 3600, "codex_7d_reset_after_seconds": 86400,
    "privacy_mode": "...", "openai_passthrough": true
  }
}
```

> 额度快照在 `extra.codex_*` 字段（Sub2API 后台已计量），`usage_bridge` 读它展示席位额度；区别于 `converter_core` 实时打 `wham/usage`。

### 2.2 导入后统一配置（bulk-update extra）

导入/建号成功后调用 #6 把账号设 `status=active`、绑分组、并设：
```jsonc
{"openai_passthrough": true,
 "openai_oauth_responses_websockets_v2_enabled": true,
 "openai_oauth_responses_websockets_v2_mode": "passthrough",
 "codex_cli_only": true}  // codex_cli_only 仅 OAuth 建号链路有
```

## 3. OIDC 卡密服务 API（`webui/oidc_client.py`）

**基础约定**：Base = `SUB2API_OIDC_API_URL`；鉴权 `Authorization: Bearer <SUB2API_OIDC_API_KEY>` + `Content-Type: application/json`；timeout 15s；**错误以数据形式返回不抛异常**。

| 函数 | 端点 | 方法 | 请求体 | 用途 |
|------|------|------|--------|------|
| `oidc_status` | `/api/status` | GET | — | 探活（`/api/oidc/test` 调用） |
| `oidc_generate_cards` | `/api/cards/generate` | POST | `{count, expires_days, note}` | 批量发卡 |
| `oidc_lookup_card` | `/api/cards/lookup` | POST | `{card}` | 查卡 |

自动维护 Phase 6 调用：`POST {OIDC}/api/cards/generate {count:N, expires_days:30, note:"auto_maintenance"}`。

> OIDC 服务本身（PHP）的完整接口与 ChatGPT Business 自定义 OIDC 登录端点见 `oidc/docs/`。

## 4. 鉴权方式对比

| 系统 | 鉴权位置 | 凭据 | 来源 |
|------|----------|------|------|
| ChatGPT accounts API | `authorization: Bearer` 头 | access_token | 母号 ACC |
| ChatGPT organizations API | `cookie` + `X-Csrf-Token` | session_token | HAR 提取 |
| ChatGPT 注册 | sentinel token + cookie + PKCE | 动态生成 | 注册流程 |
| Sub2API | `x-api-key` 头 | admin_api_key | 配置 |
| OIDC | `Authorization: Bearer` 头 | oidc_api_key | 配置 |

---

## 5. 网络出口与代理

- ChatGPT 相关请求（注册/席位/额度）建议走 `SUB2API_OUTBOUND_PROXY_URL`（SOCKS5），分散出口 IP 避免关联封号。
- Sub2API / OIDC 请求是否走代理同样取决于环境变量（`http_client` 仅 SOCKS5）。
- 注册引擎（curl_cffi）支持 socks5/http 全协议代理，每个账号绑同一出口。

---

## 小结

- ChatGPT 三套接口：注册流程、accounts 席位（Bearer）、organizations 成员（cookie）、额度/续期（wham/usage + oauth/token）。
- Sub2API 18 个管理端点，`x-api-key` 鉴权，`{code,message,data}` 响应壳；额度在 `extra.codex_*`。
- OIDC 3 个卡密端点，Bearer 鉴权，错误以数据返回。
