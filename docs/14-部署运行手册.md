# 14 · 部署运行手册（自部署完整跑通）

> 本手册基于 **`feat/javoo` 分支**（注册链路已重写为 **codex oauth + OIDC 卡密 SSO**，commit `8a65f20` 起）。
> 旧文档 04/13 描述的「企业 SSO 免验证码（`team.edu.sixoner.com` / authentik）」方案**已废弃**，以本手册为准。
> 目标：照此从零把 WebUI 服务完整跑起来并验证全链路。

---

## 0. 整体链路（先建立认知）

```
母号(ChatGPT Business team) ──Custom OIDC──> 自建 OIDC 卡密服务(oidc.xxx.com)
        │                                            ▲
        │ 自动维护调度器(WebUI)                        │ 发卡 /api/cards/generate
        ▼                                            │
  [1]改低额度号为Codex → [2]扫Sub2API → [3]判阈值 → [4]注册新号 → [5]导入Sub2API → [6]发卡
                                              │
                       codex oauth + 卡密SSO（免手机验证）拿 codex refresh_token
```

自动注册一个号的真实路径（已验证）：
**发卡 → codex auth_url(login_hint) → 母号 SSO connection → OIDC 卡密页首次激活 → signin-consent → workspace/select → callback code → 换 codex token → 导入 Sub2API**。

## 1. 前置条件

| 前置 | 说明 |
|---|---|
| OIDC 卡密服务已部署 | 如 `https://oidc.xxx.com`，已安装并能发卡。见 `oidc/` 与 `docs/oidc/`。 |
| 母号已接入 OpenAI Custom OIDC | 母号在 `admin.openai.com/identity` 验证了**邮箱域名**、Set up SSO→Custom OIDC 指向你的 OIDC 服务、connection active、SSO=Optional。见 `docs/oidc/mother-accounts/`。 |
| 母号 ACC 凭据 | 母号的 `access_token` + `account_id`（从母号 ChatGPT 登录态取，见 §3）。 |
| 干净出站代理 | **美国住宅 socks5**，全程用于注册/导入出站。注册对 Cloudflare 后端要求高质量 IP。 |
| Python | 3.11+（实测 3.11）。 |

## 3. 配置 `.env`（必填项 + 值来源）

复制模板后填写：`cp .env.example .env`

| 变量 | 必填 | 值 / 来源 |
|---|:--:|---|
| `SUB2API_BASE_URL` | ✅ | Sub2API 网关地址，如 `https://onebool.com` |
| `SUB2API_ADMIN_API_KEY` | ✅ | Sub2API 管理员 API Key |
| `SUB2API_GROUP_IDS` | | 导入账号绑定分组，如 `4` |
| `ACC_MOTHER_ACCOUNT_EMAIL` | ✅ | 母号邮箱（如 `raochris430@gmail.com`） |
| `OPENAI_ACCESS_TOKEN` | △ | 母号 access token（见下方获取方式） |
| `OPENAI_ACCOUNT_ID` | ✅ | 母号 workspace account_id（注册时用作 `workspace_id`） |
| `SUB2API_OIDC_API_URL` | ✅ | OIDC 服务地址，如 `https://oidc.1bool.com` |
| `SUB2API_OIDC_API_KEY` | ✅ | OIDC 的 **cards_api_key**（发卡用） |
| `SUB2API_AUTO_REGISTER_DOMAIN` | ✅ | **母号在 OpenAI 验证并配了 Custom OIDC 的邮箱域名**，如 `ai.1bool.com`。⚠️ 见 §6 约束① // 不是旧的 `team.edu.sixoner.com` |
| `SUB2API_OUTBOUND_PROXY_URL` | ✅(实际) | 出站代理，**建议 `socks5h://user:pass@host:port`**。见 §6 约束② |
| `SUB2API_AUTO_REGISTER_COUNT` | | 目标水位（补到数量，1~20），默认 3 |
| `SUB2API_AUTO_REGISTER_THRESHOLD` | | 补号触发阈值，默认 1 |
| `SUB2API_WEB_SECRET` | | WebUI 登录密码（**空=免登录**，公网必设） |
| `SUB2API_WEB_HOST` / `SUB2API_WEB_PORT` | | 默认 `0.0.0.0` / `28463` |
| `SUB2API_AUTO_POLICY_ENABLED` | | 自动调度开关，默认 `true` |
| `SUB2API_AUTO_POLICY_RUN_ON_START` | | 启动即跑一次，默认 `true` |
| `SUB2API_SETUP_DONE` | | 安装完成标志；6 必填齐 + 此项 `true` 调度器才会真正执行 |

> △：`OPENAI_ACCESS_TOKEN` 与 `OPENAI_SESSION_TOKEN` 至少一个。WebUI 安装页粘贴「母号 ACC 原文」后会自动写入 `OPENAI_*`。

**获取母号 access_token / account_id**：在母号已登录的浏览器（推荐指纹浏览器走同一代理）打开 `https://chatgpt.com/api/auth/session`，复制返回 JSON 里的 `accessToken`；`account_id` 可从该 token 的 JWT（`https://api.openai.com/auth.chatgpt_account_id`）解出，或从 ChatGPT 后台。

### 调试期建议
首次手动分步验证时，先关自动调度，避免一启动就跑全链路改母号席位：
```ini
SUB2API_AUTO_POLICY_ENABLED=false
SUB2API_AUTO_POLICY_RUN_ON_START=false
```
验证通过后再置 `true`。

## 4. 启动 WebUI

```bash
python entry.py --host 0.0.0.0 --port 28463
# 或后台并落日志
python entry.py --host 0.0.0.0 --port 28463 > webui.log 2>&1 &
```

- 浏览器开 `http://服务器IP:28463/`；未完成安装会进安装向导；`SUB2API_WEB_SECRET` 非空则先登录。
- 启动后调度器线程常驻：`SETUP_DONE=true` 且必填齐 → 启动 10s 后跑第一次维护；否则状态显示「等待配置」。
- 看板：`GET /api/tasks/list` 含 `scheduler` 状态与近期任务。

## 5. 自动维护 6 阶段（`run_auto_maintenance`）

| 阶段 | 动作 | 副作用 |
|---|---|---|
| 1 seat_policy | 扫 Sub2API 用量，低额度号 Sub2API 停用 + 母号席位降 Codex；ChatGPT 席位收敛 ≤2 | **改母号现有 member 席位**（只降级不删） |
| 2 remote_scan | 扫 Sub2API 账号池（alive/dead/no_quota） | 只读 |
| 3 阈值判定 | `quota_ok ≥ threshold` 则跳过补号 | — |
| 4 register | `register_batch` 注册新号拿 codex token | **真实建号**（占母号席位、消耗卡） |
| 5 import | `import_to_sub2api_codex_session` 导入 Sub2API | **写生产 Sub2API** |
| 6 oidc_cards | `oidc_generate_cards` 发卡 | 生成卡密 |

> ⚠️ Phase 1 会处理 **Sub2API 全部账号**（不止测试号）。生产环境首次跑前先确认 Sub2API 池子里没有不该被降级/停用的真实号。

## 6. 🔴 关键约束（必读，违反则注册必败）

1. **注册域名 = 母号 SSO 域名**：`SUB2API_AUTO_REGISTER_DOMAIN` 必须是母号在 OpenAI 验证并绑定了 Custom OIDC connection 的域名（如 `ai.1bool.com`）。用未绑 SSO 的域名 → OpenAI 落到 `/create-account/password`（密码+邮件验证码），本项目无收码能力 → 失败。
2. **代理用 `socks5h://`**：访问 Cloudflare 后端（chatgpt/auth.openai/sentinel）必须代理侧 DNS。`socks5://`（本地 DNS）会 TLS reset。`register.py` 已对注册链路自动把 `socks5://` 升级为 `socks5h://`，但建议 `.env` 直接写 `socks5h://`。
3. **直接 codex oauth、不碰 chatgpt 网页**：注册第一个请求就是 codex auth_url。**若先用 chatgpt.com 网页登录再 codex oauth，会触发 `add-phone` 手机验证。** 自动注册天然满足（纯 HTTP），手动调试切忌先开 chatgpt 网页。
4. **一个号只能用一次**：同一邮箱第二次走 codex oauth 必触发 add-phone。注册失败**不要用同邮箱重试**，换新前缀。
5. **永不 remove member**：账号「清理/降级」一律是把**席位类型改为 Codex**（`usage_based`），保留 member。架构上 `/api/acc/seat` 也只允许改 Codex。

## 7. 逐环节验证（手动分步，推荐首次）

> 均在项目根、激活 venv 后执行。只读项安全；注册/导入为真实写。

```bash
# (a) Sub2API 连接 / OIDC 状态（只读）
python - <<'PY'
from newtoken.webui.config import WebState, ENV_PATH
from newtoken.sub2api.remote import test_sub2api_connection
from newtoken.webui.oidc_client import oidc_status
s=WebState(ENV_PATH)
print("sub2api:", test_sub2api_connection(s.build_remote_config()))
print("oidc   :", oidc_status(s.load_config()))
PY

# (b) 母号成员（只读）
python - <<'PY'
from newtoken.webui.config import WebState, ENV_PATH
import newtoken.acc.seat_client as sc
s=WebState(ENV_PATH); c=s.build_seat_client()
for u in sc.list_all_users(c):
    print(u.get("email"), u.get("seat_type"), u.get("role"))
PY

# (c) 注册一个号（真实写：建号+拿 codex token）
python - <<'PY'
from newtoken.webui.config import WebState, ENV_PATH
from newtoken.webui.register import register_batch
s=WebState(ENV_PATH); v=s.load_config()
r=register_batch(1, email_domain=v["SUB2API_AUTO_REGISTER_DOMAIN"],
  proxy_url=v["SUB2API_OUTBOUND_PROXY_URL"], oidc_api_url=v["SUB2API_OIDC_API_URL"],
  oidc_api_key=v["SUB2API_OIDC_API_KEY"], account_id=v["OPENAI_ACCOUNT_ID"], max_workers=1)
for x in r: print(x.ok, x.email, x.steps_completed, x.error)
PY
```

- 注册成功标志：`ok=True`，`steps` 含 `codex_token`，单号约 30s。
- 完整一键：WebUI 触发 `auto_maintenance` 任务，或脚本调 `run_auto_maintenance(state)`。

## 8. 故障排查

| 现象 | 原因 / 处理 |
|---|---|
| 注册 0.x s 即 `Recv failure / Connection reset` | 代理 `socks5://` 本地 DNS，改 `socks5h://`；或代理 IP 质量差/被风控，换 IP |
| `unexpected authorize destination: .../create-account/password` | 注册域名未绑母号 SSO → 改 `SUB2API_AUTO_REGISTER_DOMAIN` 为母号验证域名 |
| `SSO approve form missing challenge` | 旧 authentik 流残留判断；现行走 OIDC 卡密，确认用的是 feat/javoo 新 `register.py` |
| 落到 `auth.openai.com/add-phone` | 先碰了 chatgpt 网页登录，或该号已用过一次 → 换新号、纯 codex oauth |
| `workspace/select` 非 200 / 无 continue_url | 检查 `OPENAI_ACCOUNT_ID` 是否为母号 workspace id；母号 token 是否过期 |
| 发卡 403 | OIDC 经 Cloudflare，确认请求带正确 UA（代码已用 curl_cffi 直连）；Key 是否为 `cards_api_key` |
| ACC 席位接口偶发 reset | 母号侧出站也会间歇 reset；重试即可（建议后续给 seat_client 加重试） |

## 9. 测试 / 清理（永不删 member）

把测试号或低额度号「清理」= **改席位为 Codex**：
```bash
python - <<'PY'
from newtoken.webui.config import WebState, ENV_PATH
import newtoken.acc.seat_client as sc
s=WebState(ENV_PATH); c=s.build_seat_client()
users=sc.list_all_users(c)
for u in users:
    if u.get("seat_type")=="default" and u.get("role")=="standard-user":  # ChatGPT 席位的普通成员
        sc.ensure_user_seat(c, user_id=u["id"], email=None, target_seat_type=sc.CODEX_SEAT_TYPE)
        print("→ Codex:", u.get("email"))
PY
```
> `usage_based` = Codex 席位，`default` = ChatGPT 席位；owner 不动。

## 10. 上线建议

- `SUB2API_WEB_SECRET` 设强密码；用 Nginx 反代 HTTPS，设 `SUB2API_WEB_PUBLIC_BASE_URL`。
- 母号 token 会过期（access token 通常数日），到期需更新 `OPENAI_ACCESS_TOKEN`。
- 维护硬编码版本号（`CLIENT_BUILD_NUMBER`/`OAUTH_CLIENT_ID`/`SENTINEL_SDK`/Chrome 指纹），OpenAI 改版需同步，见 [13](./13-已知问题与维护要点.md)。
- 关注调度看板各阶段 ok/fail；注册失败多先查代理质量与注册域名 SSO 状态。
