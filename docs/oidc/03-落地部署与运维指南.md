# 多母号 OIDC 落地部署与运维指南

> 配套文档：[01-设计方案](./01-设计方案.md)、[02-实施计划](./02-实施计划.md)

## 一、这是什么

一套自研 PHP OIDC 服务（基于 `oidc/`），让**单一服务**为**多个** OpenAI ChatGPT Business「team 母号」做 SSO。每个母号 = 独立 `client_id`/`client_secret` + 独立邮箱域名；终端用户用**卡密 + 邮箱前缀**自助登录。用户/卡密全局共享、不隔离。

## 二、架构要点

- 单 OIDC 服务域名（issuer），所有母号共用 discovery / jwks / authorize / token / userinfo。
- 母号靠 `oidc_clients` 表区分，允许域名靠 `oidc_client_domains` 表（`domain_normalized` 全局唯一索引）。
- `iss` 全局一致、`aud` 为各母号 `client_id`、callback 各自唯一——符合 OIDC「一个 IdP 多 client」标准。

## 三、环境要求

- PHP 7.4+（已在 8.5 实测通过），扩展：`pdo_mysql` / `openssl` / `intl`（域名归一化 `idn_to_ascii` 需 intl）。
- MySQL 5.7+。
- 宝塔站点目录指向 `oidc/public`，HTTPS。

## 四、部署步骤

1. 上传 `oidc/` 到服务器（如 `/www/wwwroot/gpt-oidc`）。
2. 宝塔站点目录指向 `oidc/public`，PHP 7.4+，开 HTTPS。
3. 伪静态：所有请求重写到 `index.php`，禁止访问 `app/` `sql/` `storage/`（见 `oidc/DEPLOY_OIDC.md`）。
4. 创建并赋写权限：`storage/keys`、`storage/exports`、`storage/ratelimits`（`chown www`）。
5. 访问 `https://你的域名/install` 走安装向导，填 app 地址 / 数据库 / 首个管理员。安装会自动生成 `app_key`/`app_pepper`/`cards_api_key`/`clients_admin_api_key` + RSA 签名密钥 + 建表。
   - ⚠️ 安装向导里残留的单 client 输入框可**留空**（已废弃，新逻辑会忽略）。

## 五、配置项（`app/config.php`）

| 配置 | 说明 |
|---|---|
| `cards_api_key` | WebUI 发卡/查询 API（`/api/cards/*`、`/api/status`） |
| `clients_admin_api_key` | 母号管理 API（`/api/clients`，高权限，独立 key） |
| `clients_admin_ip_allowlist` | 母号管理 API 的 IP 白名单（空=不限）；用 `REMOTE_ADDR` 校验，**需服务在剥离转发头的可信反代后才有意义** |
| `clients_secret_reveal_enabled` | 是否允许经 API 解密回显 secret（默认 `false`） |
| `oidc_issuer` | OIDC issuer，所有母号共用 |

## 六、新增一个母号 SOP（核心运维流程）

1. **OpenAI**（`admin.openai.com/identity`）验证该母号域名：加 DNS TXT；子域名需各自单独验证。
2. **本服务后台**「母号管理」→ 新建母号：填名称 + 允许域名（回调可暂留空）→ 生成 `client_id`/`client_secret`（**仅展示一次，立即复制**）。
3. **OpenAI** → Set up SSO → Custom OIDC：填本服务 issuer/discovery + 该母号 `client_id`/`client_secret`。
4. OpenAI 生成 **Login redirect URI（callback）** → 回后台编辑该母号，填入「回调白名单」。
5. 建议先用 OpenAI **测试沙箱**验证；生产保持 SSO=Optional + 开两个浏览器窗口防锁死 → 发对应域名卡密。

> **域名约束**：一个域名只能归一个母号（OpenAI 硬约束 + 本服务唯一索引双保险）。多母号必然多域名——子域名切分（一主域名切 `m1.`/`m2.`…各自 TXT）与独立顶级域名可混用。邮箱无需真实收件。

## 七、OpenAI 配置值（每母号填）

| 字段 | 值 |
|---|---|
| Issuer | `https://你的域名` |
| Discovery URL | `https://你的域名/.well-known/openid-configuration` |
| Client ID / Secret | 后台该母号生成的值 |
| Scopes | `openid profile email` |

必需 claims：`sub / email / given_name / family_name`（已内置于 id_token 与 /userinfo）。

## 八、已验证测试矩阵（本地 PHP 8.5 + MySQL 实测通过）

- 纯逻辑单测 **11/11**：域名归一化、AES-256-GCM 加解密、redirect 白名单。跑：`php oidc/cli/tests/run.php`。
- 端到端 `authorize → /sso 卡密激活 → resume → /token`：id_token `aud`/`email`/`iss`/`nonce`/姓名拆分均正确，RS256 签发。
- 首次激活护栏、域名越界拒绝、多 client 串卡拒绝、disabled 母号拒绝。
- API 分权（cards key 不能管母号）、reveal 默认禁用、secret 不外泄、域名唯一、update 事务回滚、`/api/status`、rotate+字段同时生效。

## 九、安全要点

- `client_secret` AES-256-GCM 加密存储（`app_key` 派生），后台显示需二次确认 + 审计。
- `cards_api_key` 与 `clients_admin_api_key` 分离（最小权限）；WebUI 只持发卡 key 也读不到母号 secret。
- `clients_admin_ip_allowlist` 用 `REMOTE_ADDR`——仅在服务位于剥离 `X-Forwarded-For`/`CF-Connecting-IP` 的可信反代后才可靠。
- 卡密**首次激活必须从 OpenAI 发起**（pending client 上下文），堵住直接 `/sso` 旁路。
- **残余风险**：终端用户若已知多个母号入口，理论上仍可把未绑定卡绑到另一母号域名（卡-母号未硬绑定，属 YAGNI 取舍）；靠运营按域名发卡约束。

## 十、故障排查

| 现象 | 原因 / 处理 |
|---|---|
| 全站 302 跳 `/install` | installRecovery：已配置 + 有表但无管理员 → 重建管理员或走 `/install` 恢复 |
| `/authorize` 报 redirect_uri 无效 | 该母号回调白名单未填 OpenAI 给的 callback → 后台编辑母号填入 |
| 建母号报「域名已被占用」 | 该域名已属其它母号（唯一索引）→ 换域名或先从原母号移除 |
| `sso_error` / 看似没权限 | 邮箱为空或域名未在 OpenAI 验证 → 确保 email 非空且域名已 DNS TXT 验证 |
| id_token 签名错 | `storage/keys` 私钥缺失/不可读 → 重新生成签名密钥 |
